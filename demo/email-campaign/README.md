# Email campaign demo (presenter)

Two packaged handoff folders live next to this file. They are **too deep** for the automatic repo scan (depth limit), so the demo relies on **drag-and-drop** (or **Add folder…**) into Agent Brief.

1. In the app, pick **Email · thin handoff** or **Email · strong handoff** so the task and intro steps match the story.
2. From Finder / Explorer, drag **`handoff-sparse`** or **`handoff-rich`** (the folder itself) onto the workspace drop zone, or use **Add folder…**.
3. Run **Run Pre-flight Check** and compare how risks change between the thin vs strong handoff.

Clear uploaded files before switching between thin and strong so the model is not mixing both sets of context.
